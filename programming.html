Core Java

Abstract Class ve Interface Farkı Nedir?

Abstract Classlar base class olarak kullanılan yapılardır. Abstract classlardan nesne örneği yaratılamadığından dolayı zorunlu olarak başka classlara kalıtım vermek zorundadırlar.

  • Kalıtım: Bir class sadece bir tane abstract classtan miras alabilirken birden fazla interface’i implement edebilir.
  • Gövde: Abstract classların içlerinde gövdelere kod yazılabilirken interface’lerde sadece imzalar oluşturulabilir (Java 8+ default metotlar hariç).
  • Amaç: Abstract classlar "is-a" ilişkisi (temel yapı) için, interfaceler "can-do" ilişkisi (yetenek/sözleşme) kazandırmak için kullanılır.

Abstract classların içerisindeki abstract memberların, inherit edildiği yerde override edilmesi zorunludur.

Core Java

== ile .equals() Farkı Nedir?

Java'da eşitlik kontrolü iki seviyede yapılır: Referans ve İçerik.

  • == Operatörü: Bellekteki referansları (adresleri) karşılaştırır. Primitive tiplerde ise doğrudan değeri karşılaştırır.
  • .equals() Metodu: Nesnelerin içeriğini karşılaştırır. Override edilmezse referans karşılaştırması yapar.
String s1 = new String("Merhaba"); String s2 = new String("Merhaba"); System.out.println(s1 == s2); // false (Biri Heap'te diğeri Pool'da olabilir) System.out.println(s1.equals(s2)); // true (İçerik aynı)

Core Java

Static ve Final Anahtar Kelimeleri

static

Sınıfa ait olan (instance'a değil) öğeleri tanımlar. Bellekte sadece bir defa oluşturulur (Method Area/Metaspace).

final

Değiştirilemezlik belirtir. Değişkende değeri sabitler, metotta override'ı engeller, sınıfta kalıtımı (inheritance) durdurur.

public static final double PI = 3.14159; // Global bir sabit

JVM Internals

Garbage Collector & Memory Management

Stack vs Heap

  • Stack: Metod çağrıları, local değişkenler ve referanslar burada tutulur (LIFO - Çok Hızlı).
  • Heap: Nesneler (objects) ve sınıf değişkenleri burada saklanır (Dinamik - Daha Yavaş).

Garbage Collector

JVM tarafından yönetilen, kullanılmayan nesneleri bellekten otomatik temizleyen mekanizmadır. G1 GC, ZGC ve Shenandoah modern algoritmalar arasındadır.

Concurrency

Serializable & Thread Safety

Thread Safe

Çoklu iş parçacığı (multithreaded) ortamında bir nesneye aynı anda erişildiğinde verinin bozulmamasıdır. synchronized, volatile ve Atomic sınıflar ile sağlanır.

Serializable

Nesnenin byte dizisine (binary) çevrilerek ağ üzerinden taşınabilmesini veya diske yazılabilmesini sağlar. Kimlik (ID) taşımayan, sadece veri tutan nesnelere ise Value Object denir.

AOP

@Transactional & AOP (Aspect Oriented Programming)

AOP, kod tekrarını azaltmak için "cross-cutting concerns" (kesen ilgiler) denilen loglama, güvenlik ve transaction işlemlerini modüler hale getirir.

@Transactional Yayılımı (Propagation)

  • REQUIRED: Mevcut transaction'ı kullanır, yoksa yeni açar.
  • REQUIRES_NEW: Her zaman yeni bir transaction açar, öncekini bekletir.
  • NESTED: Mevcut içinde checkpoint oluşturur.

Fault Tolerance

Circuit Breaker ve Resilience4j

Sistem arızalarını tespit edip "sigorta" gibi davranarak çöküşü (cascading failure) engeller.

Durumlar (States)

  • Closed: Her şey normal, akış devam ediyor.
  • Open: Hatalar eşiği aştı, istekler anında reddediliyor (Fallback çalışır).
  • Half-Open: Sistem sınırlı sayıda istekle test ediliyor.

Resilience4j; Retry, Rate Limiter ve Bulkhead gibi ek dayanıklılık modülleri sunar.

Best Practices

GOF Design Patterns

Yazılım geliştirmede sık karşılaşılan sorunlara getirilen standart çözümlerdir. 3 ana kategoriye ayrılırlar:

  • Creational (Kurucu): Nesne oluşturma mantığını gizler ve esneklik sağlar. (Singleton, Factory, Builder, Prototype).
  • Structural (Yapısal): Sınıf ve nesne yapılarının komposizyonunu düzenler. (Adapter, Facade, Decorator, Proxy).
  • Behavioral (Davranışsal): Nesneler arası iletişim ve sorumluluk dağılımını belirler. (Strategy, Observer, State, Command).

Database Strategy

DB Sharding ve Partitioning

Partitioning: Büyük tabloları fiziksel olarak parçalara bölmektir (Range, List, Hash).

Sharding: Yatay bölümleme (Horizontal Partitioning) yaparak veriyi farklı sunuculara (shard) dağıtmaktır. Performans ve ölçeklenebilirlik sağlar.

Ayrıca PL/SQL ile veritabanı içinde prosedürel işlemler ve triggerlar yönetilir.

Search Speed

Indexing ve B-Tree Mekaniği

İndeks, tablonun tüm satırlarını taramak (Full Table Scan) yerine, bir sözlük gibi veriye doğrudan ulaşmayı sağlar.

  • B-Tree Index: Sıralı, dengeli bir ağaçtır. Aralık (range) sorguları için idealdir.
  • Hash Index: Sadece eşitlik (=) sorguları için en hızlıdır.

Aşırı indeksleme; INSERT/UPDATE işlemlerini yavaşlatır çünkü her değişimde indeks de güncellenmelidir.

ACID Prensipleri ve Normalizasyon

ACID

  • Atomicity: Hepsi ya da hiçbiri.
  • Consistency: Verinin tutarlı bir state'te kalması.
  • Isolation: İsteklerin birbirinden izole çalışması.
  • Durability: Onaylanan verinin kalıcılığı.

Performance

TPS, Deadlock ve Bottleneck

Principles

SOLID Yazılım Prensipleri

Esnek, bakım yapılabilir ve ölçeklenebilir kod yazmak için 5 temel kural:

  • S - Single Responsibility: Bir sınıfın sadece bir görev yüklenmesi.
  • O - Open/Closed: Gelişime açık, değişime kapalı olma.
  • L - Liskov Substitution: Alt sınıfların üst sınıfların yerine geçebilmesi.
  • I - Interface Segregation: Özelleşmiş küçük interfaceler oluşturma.
  • D - Dependency Inversion: Üst seviye modüllerin alt seviye modüllere değil, abstraction'lara bağımlı olması.

Architecture

gRPC (Google Remote Procedure Call) Nedir?

gRPC, servisler arası iletişimi (Service-to-Service communication) modernize eden, yüksek performanslı ve açık kaynaklı bir RPC protokolüdür. Özellikle mikroservis mimarilerinde iç ağ trafiğini minimize etmek için kullanılır.

Neden gRPC?

  • Protocol Buffers (Protobuf): JSON gibi metin tabanlı formatlar yerine binary (ikili) serileştirme kullanır. Bu sayede veriler çok daha küçük boyutta taşınır ve CPU tarafından daha hızlı işlenir.
  • HTTP/2 Tabanlı: Multiplexing özelliği sayesinde tek bir bağlantı üzerinden aynı anda birçok istek gönderilebilir. Ayrıca Header Compression ile bant genişliği tasarrufu sağlar.
  • Strongly Typed: Bir .proto dosyası ile servislerin kontratı (arayüzü) önceden belirlenir. Bu, hata payını minimize eder.
  • Streaming: Çift yönlü (bi-directional) veri akışını destekleyerek anlık veri iletimini kolaylaştırır.
